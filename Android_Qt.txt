Android NDK (длЯ разработки на ‘++) такой же родной как как и Android SDK (длЯ разработки на Java).
‚ крайнем случае - может потребоватьсЯ стыковка с джава-объектами, и тут вам потребуетсЯ изучать Android NDK, и возможно даже писать немного оберточного кода на Java.
1. Qt - это всЮ-таки C++. ђазрабатывать на нЮм существенно сложнее, чем на Java. ’о есть дольше и с большим количеством ошибок.
3. Java - роднаЯ платформа длЯ Android. Ћтсюда потенциальные проблемы с совместимостью у Qt.
Ђ когда вы пишете под Android на Java - вы всегда пишете кусок кода работающий в колнтексте DalvikVM и должны жить по предусмотренным данной машиной паттернам и сценариЯм...
…щЮ к слову про миф "на Java писать проще чем на C++"(если сравниваетсЯ Java под DalvikVM и C++\Qt5): Ќе забывайте - DalvikVM - это вам не JavaVM.
Ђ когда вы пишете на Qt - у вас гарантирована поддержка единых механизмов длЯ всех платформ.
Ќо из Qt - может быть сложно поиметь доступ к сугубо-андроидным сервисам типа поставщика данных.
